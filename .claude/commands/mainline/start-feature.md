작업명(한글)을 받아 feature 브랜치를 생성하고 전환한다.

인자: $ARGUMENTS (한글 작업명, 예: 로그인 검증 추가)

**기준 브랜치** = 팀 규약의 통합 브랜치(기본 `develop`). `develop`이 없으면 현재 기본 브랜치(`main` 등)를, 확정되지 않았으면 사용자에게 확인해 정한다. 이하 `<기준>`으로 표기한다.

작업 절차:
1. `<기준>` 브랜치를 확정한다(위 규칙).
2. $ARGUMENTS 한글 작업명을 영문 kebab-case slug로 변환한다 (예: "로그인 검증 추가" → "add-login-validation").
3. 현재 브랜치 상태를 확인한다 (커밋되지 않은 변경이 있으면 경고).
4. 원격(`origin`)이 있으면 `<기준>`을 최신으로 pull 한다. 원격이 없거나 프로젝트 생성 전이면 로컬 `<기준>` 기준으로 진행하고 그 사실을 알린다.
5. `<기준>`을 기준으로 feature/{slug} 브랜치를 생성하고 전환한다.
6. 결과를 출력한다.

출력 형식:
- 입력된 작업명 (한글)
- 변환된 브랜치명
- 기준 브랜치와 최신 커밋
